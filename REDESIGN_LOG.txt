6. THREE.JS SIGNAL FIELD
================================================================================

File:           app/components/HeroField.tsx
Placement:      / hero, right half of the xl: two-column layout
Lazy-load:      dynamic(() => import('./components/HeroField'),
                        { ssr: false, loading: HeroFieldSkeleton })
Bundle size:    117KB gzipped (budget: 150KB)

Scene:
  22 horizontal polylines (three.Line), 96 points each.
  Y displacement per frame via layered pseudo-noise (3 sin/cos waves).
  Cursor influence: smoothstep radial bump, radius 0.42, strength 0.12.
  Each row's opacity fades toward top/bottom edges (center-weighted).
  Colour sampled from --color-ink (#16181d) with per-row alpha 0.14-0.36.

Performance:
  OrthographicCamera (flat 2.5D)
  powerPreference: 'low-power'
  pixelRatio clamp min(2, devicePixelRatio)
  ResizeObserver for container size
  IntersectionObserver threshold 0.01, rootMargin 200px (pauses off-screen)
  document.visibilitychange (pauses when tab hidden)
  prefers-reduced-motion: renders a single static frame, no RAF

Fallback tree:
  1. prefers-reduced-motion: reduce -> static single frame
  2. no WebGL: returns silently; the container still renders the CSS
     radial-gradient placeholder behind it
  3. mobile sizing: aspect-[4/3] on base, aspect-[5/4] at md,
     min(600px, 80vh) at xl

Cleanup (on unmount):
  - cancelAnimationFrame
  - disconnect ResizeObserver + IntersectionObserver
  - remove visibilitychange listener
  - remove pointermove + pointerleave listeners
  - dispose every row's BufferGeometry + LineBasicMaterial
  - group.clear() + scene.clear()
  - renderer.dispose()
  - remove canvas from DOM
